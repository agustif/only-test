only-test runner policy for Bun-heavy repos

Choose one runner. Let `bun test` follow policy.

`only-test` is the generic line: a repo declares one approved runner, the global Bun shim reads that policy, and bare `bun test` is redirected without developers needing to remember which command path is safe.

curl -fsSL https://agustif.github.io/only-test/install.sh | bash

Supported in this release

bun-test

Keep Bun’s own runner, but make the choice explicit and consistent across the repo.

vitest

Redirect bare `bun test` to the local Vitest binary resolved from the repo.

jest

Use the same Bun interception mechanism while standardizing on Jest instead.

Default shape

Minimal repo config, global shim by default, optional script syncing, optional local shim when you need hermetic behavior.

Minimal repo state

The preferred path stores policy in `package.json#onlyTest` or `only-test.json` instead of littering the repo with enforcement artifacts.

Global enforcement

The machine-level Bun shim lives under `~/.local/share/only-test/`, so one setup step can protect many repos.

Opt-in repo shims

If a team needs local-only enforcement for CI or a hermetic workspace, the repo-local shim path is still available.

01

Declare policy

only-test init --runner vitest
02

Install the shim

only-test setup-shell
03

Activate once

eval "$(only-test activate)"
